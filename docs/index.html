## features

Built for the set you're actually in.

// card-ui

One card, one set

Each set is a focused card: exercise, reps or duration, per-side flag, and notes. No scrolling mid-lift.

// timers

Split rest timers

Separate rest between sets and after the final set. Progress ring, skip control, haptic at zero.

// ratings

Fail / Good / Easy

Rate each set in one tap. Enough Easies progress you; enough Fails pull the weight back.

// summary

Obsidian-shaped output

End-of-workout markdown summary copies to the clipboard. All-good days stay short; exceptions stand out.

// json

JSON in, JSON out

Import routines straight from an LLM-generated JSON blob. Export anything you've built in the editor.

// theme

TokyoNight, always

Monospaced type, bracketed controls, comment-style headers. It looks like a coding session because it is one.

## flow

From home screen to clipboard markdown.

01

Pick a routine

Starter routines seed on first launch. Edit freely or paste JSON.

02

Work the sets

Swipe to complete reps. Timed exercises auto-advance at zero.

03

Rest, then repeat

Countdown ring, next-up label, vibration when the timer ends.

04

Copy the summary

Markdown summary with adjustments and flags, ready for your log.

## data

Routines are just JSON.

No backend, no account, no lock-in. Routines live as JSON in the app's documents directory. Build them in the editor, export to your notes, or have an LLM draft one and paste it in.

// backward-compatible decoding handles older fields and legacy rest values. 

// Routine.json
{
  "name": "Push Day",
  "sections": [{
    "name": "Chest & Shoulders",
    "exercises": [{
      "name": "Incline Press",
      "sets": 4,
      "reps": 8,
      "restBetweenSetsSeconds": 60,
      "restAfterExerciseSeconds": 120,
      "perSide": false
    }]
  }]
}
